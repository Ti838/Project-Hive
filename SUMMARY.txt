TECHNOLOGY STACK (CURRENT)
================================================================================

  FRONTEND
    - HTML5, CSS3, Vanilla JavaScript (ES2020+)
    - No framework, no build step — pure static files
    - Served by Vercel (CDN, auto-deploy on git push)
    - Socket.IO v4 Client (real-time)
    - Google Fonts (Inter), Google Material Symbols
    - Cloudflare Turnstile CAPTCHA

  BACKEND
    - Node.js 18+ / Express.js 4.x
    - Socket.IO 4.x (WebSocket server)
    - JWT HS256 authentication
    - bcryptjs (password hashing, 12 rounds)
    - nodemailer + Brevo SMTP (email delivery)
    - Groq (llama-3.3-70b primary AI) + Google Gemini (fallback + vision)
    - WebRTC peer-to-peer calling engine
    - TURN/STUN relay (Metered.ca Open Relay — HMAC credentials)
    - Jitsi Meet embedded (group calls)
    - helmet, cors, express-rate-limit, morgan
    - Hosted on Render (free tier)

  DATABASE
    - Supabase PostgreSQL (Singapore region)
    - @supabase/supabase-js v2
    - Row-level security + service role for admin ops

  SERVICES
    - Supabase          — PostgreSQL database
    - Brevo SMTP        — Transactional email
    - Groq              — Primary AI engine (llama-3.3-70b)
    - Google Gemini     — Fallback AI + image analysis
    - Cloudflare        — Turnstile CAPTCHA
    - Metered.ca        — Free TURN relay server (WebRTC NAT traversal)
    - Jitsi Meet        — Group video/voice calling
    - Google OAuth      — Social login via Supabase provider
    - Vercel            — Frontend hosting (static CDN)
    - Render            — Backend hosting (Node.js)

SECURITY (21 Vulnerabilities Identified & Resolved — OWASP Top 10)
================================================================================

  AUTHENTICATION & ACCESS CONTROL
    ✅ JWT HS256 authentication (access + refresh token rotation)
    ✅ Token type validation — access/refresh tokens cannot be confused
    ✅ bcrypt password hashing (12 rounds)
    ✅ Email verification required before login
    ✅ No hardcoded fallback JWT secrets — server fails fast if missing
    ✅ Admin credentials stored only in server env (never in DB)
    ✅ Admin JWT separate from user JWT (type: admin_access, 4h expiry)
    ✅ Admin login uses timing-safe comparison (crypto.timingSafeEqual)
    ✅ Admin brute-force protection (5 attempts / 15-minute lockout)
    ✅ Dev privilege escalation endpoint disabled at route level in production

  INPUT VALIDATION & INJECTION PREVENTION
    ✅ XSS sanitization middleware strips <script>, event handlers, javascript: URIs
    ✅ SQL/PostgREST filter injection prevented via sanitizeSearch() (7 endpoints)
    ✅ SSRF protection blocks internal/private/loopback IPs on URL scraper

  INFRASTRUCTURE SECURITY
    ✅ Content Security Policy (CSP) with strict CDN allowlist via Helmet
    ✅ HSTS (max-age=31536000; includeSubDomains; preload)
    ✅ Referrer-Policy: strict-origin-when-cross-origin
    ✅ Permissions-Policy: camera=(), microphone=(), geolocation=()
    ✅ X-Frame-Options: SAMEORIGIN (prevents clickjacking)
    ✅ Cloudflare Turnstile CAPTCHA verification on auth endpoints
    ✅ Layered rate limiting: 500 req/15min global + 20 req/15min auth
    ✅ CORS configured for frontend origin only
    ✅ Body size limit capped at 2MB (prevents DoS)
    ✅ HTTPS enforced by Vercel and Render

  WEBRTC SECURITY
    ✅ DTLS-SRTP end-to-end encryption on all media (audio/video)
    ✅ Time-limited TURN credentials (24h HMAC expiry)
    ✅ Server-side friendship verification before call relay
    ✅ CSP allows only meet.jit.si for embedded group calls

  Full audit report: docs/SECURITY_AUDIT.md
